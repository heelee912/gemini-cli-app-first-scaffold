# Publication Checklist

Before publishing a fork or derivative repository:

- Remove `.env`, OAuth folders, cookies, tokens, API keys, and browser profile folders.
- Remove local account names and personal workspace paths.
- Remove company-internal facts, server addresses, deployment details, source code, and raw data.
- Keep only fictional or public-safe examples.
- Do not redistribute raw prompt corpora unless their license allows it.
- Do not redistribute copied browser-extension or native-host backend code unless you have the right to do so.
- Keep evidence screenshots if they use fictional data and do not reveal local browser profiles or account UI.
- Run a text scan for private identifiers before pushing.
